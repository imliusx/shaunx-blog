# Shaunx Blog 运维操作手册

本文档用于日常部署、配置、启动、停止、更新、备份和卸载 Shaunx Blog。

当前服务器部署约定：

```text
代码目录：/opt/shaunx-blog
数据目录：/opt/shaunx-blog/shaunx-blog-data
服务端口：3131
访问地址：http://23.80.83.80:3131
```

不要把 `.env`、`.env.local`、后台密码、JWT 密钥提交到 GitHub。

## 目录结构

服务器上主要目录如下：

```text
/opt/shaunx-blog
├── docker/
│   ├── Dockerfile
│   └── docker-compose.yml
├── .env
└── ...

/opt/shaunx-blog/shaunx-blog-data
├── content/
│   ├── posts/
│   ├── pages/
│   └── images/
└── config/
    └── site.config.json
```

说明：

- `/opt/shaunx-blog` 是代码目录，可以通过 Git 更新。
- `/opt/shaunx-blog/shaunx-blog-data` 是持久化数据目录，保存文章、页面、图片和站点配置。
- 容器删除或重建不会删除 `/opt/shaunx-blog/shaunx-blog-data`，除非手动删除该目录。

## 环境要求

服务器需要：

```bash
docker --version
docker compose version
git --version
```

如果缺少 Docker 或 Git，需要先安装。

## 首次部署

进入 `/opt`：

```bash
cd /opt
```

拉取代码：

```bash
git clone git@github.com:imliusx/shaunx-blog.git /opt/shaunx-blog
cd /opt/shaunx-blog
```

如果仓库是私有仓库，服务器需要配置 GitHub Deploy Key。

创建数据目录：

```bash
mkdir -p /opt/shaunx-blog/shaunx-blog-data/content
mkdir -p /opt/shaunx-blog/shaunx-blog-data/config
```

创建 `.env`：

```bash
cp docker/.env.example .env
chmod 600 .env
```

编辑 `.env`：

```bash
nano .env
```

建议配置：

```env
USER_ID=1001
GROUP_ID=1001
BLOG_PORT=3131
DATA_PATH=/opt/shaunx-blog/shaunx-blog-data
SITE_URL=http://23.80.83.80:3131
GITHUB_URL=https://github.com/imliusx
EMAIL=
TWITTER_URL=
REVALIDATE_SECRET=replace-with-a-long-random-string
ADMIN_PASSWORD=replace-with-admin-password
JWT_SECRET=replace-with-another-long-random-string
```

启动：

```bash
docker compose --env-file .env -f docker/docker-compose.yml up -d --build
```

查看状态：

```bash
docker compose --env-file .env -f docker/docker-compose.yml ps
```

正常状态应类似：

```text
Up ... (healthy)   0.0.0.0:3131->3000/tcp
```

## 配置说明

### 运行配置

运行配置在：

```text
/opt/shaunx-blog/.env
```

常用字段：

```text
BLOG_PORT        宿主机端口，默认 3131
DATA_PATH        持久化数据目录
SITE_URL         站点访问地址
ADMIN_PASSWORD   管理后台登录密码
JWT_SECRET       JWT 签名密钥
REVALIDATE_SECRET 配置刷新密钥
```

修改 `.env` 后需要重启容器：

```bash
cd /opt/shaunx-blog
docker compose --env-file .env -f docker/docker-compose.yml up -d
```

### 站点配置

站点配置在：

```text
/opt/shaunx-blog/shaunx-blog-data/config/site.config.json
```

常见字段：

```json
{
  "title": "Tiny Blog",
  "description": "站点描述",
  "url": "http://23.80.83.80:3131",
  "secureEntrance": "后台入口码",
  "postsPerPage": 6
}
```

后台入口地址格式：

```text
http://23.80.83.80:3131/admin?key=<secureEntrance>
```

其中 `<secureEntrance>` 来自 `site.config.json`。

可以通过后台设置页修改站点配置，也可以直接编辑 JSON 文件。直接编辑后建议重启：

```bash
cd /opt/shaunx-blog
docker compose --env-file .env -f docker/docker-compose.yml restart
```

## 本地写文章后发布

推荐把 Git 作为内容唯一来源：本地修改 `content/`，提交并推送到 GitHub，然后在服务器执行内容部署脚本。

本地流程：

```bash
cd /Users/liusx/git-workspace/shaunx-blog
# 修改 content/posts、content/pages 或 content/images
git add content
git commit -m "docs: publish new post"
git push
```

服务器发布：

```bash
cd /opt/shaunx-blog
scripts/deploy-content.sh
```

预览将要同步的文件，不实际修改：

```bash
cd /opt/shaunx-blog
scripts/deploy-content.sh --dry-run
```

脚本默认行为：

- 拉取 `origin/main` 的最新提交
- 将仓库 `content/` 同步到 `DATA_PATH/content/`
- 默认使用 `--delete`，让服务器内容与 Git 内容一致
- 重启 `blog` 容器

如果后台临时写过文章，但没有提交到 Git，执行该脚本时可能会被 Git 内容覆盖。

## 自动部署内容

可以使用 systemd timer 让服务器定时检查 GitHub。你只负责本地写文章并提交，服务器会自动拉取并同步内容。

自动部署执行逻辑：

- 每分钟运行一次 timer
- 检查 `origin/main` 是否有新提交
- 没有新提交时直接退出，不重启容器
- 有新提交时同步 `content/` 到 `shaunx-blog-data/content/`
- 同步后重启 `blog` 容器

安装 systemd service：

```bash
cat >/etc/systemd/system/shaunx-blog-content-deploy.service <<'EOF'
[Unit]
Description=Deploy Shaunx Blog content from GitHub
Wants=network-online.target
After=network-online.target docker.service

[Service]
Type=oneshot
WorkingDirectory=/opt/shaunx-blog
ExecStart=/opt/shaunx-blog/scripts/deploy-content.sh --only-on-change
Nice=5
EOF

cat >/etc/systemd/system/shaunx-blog-content-deploy.timer <<'EOF'
[Unit]
Description=Check Shaunx Blog content updates every minute

[Timer]
OnBootSec=1min
OnUnitActiveSec=1min
AccuracySec=15s
Persistent=true
Unit=shaunx-blog-content-deploy.service

[Install]
WantedBy=timers.target
EOF

systemctl daemon-reload
systemctl enable --now shaunx-blog-content-deploy.timer
```

查看自动部署状态：

```bash
systemctl list-timers --all | grep shaunx-blog
systemctl status shaunx-blog-content-deploy.timer
```

查看最近日志：

```bash
journalctl -u shaunx-blog-content-deploy.service -n 100 --no-pager
```

手动触发一次：

```bash
systemctl start shaunx-blog-content-deploy.service
```

停止自动部署：

```bash
systemctl disable --now shaunx-blog-content-deploy.timer
rm -f /etc/systemd/system/shaunx-blog-content-deploy.timer
rm -f /etc/systemd/system/shaunx-blog-content-deploy.service
systemctl daemon-reload
```

## 内容管理

后台管理地址：

```text
http://23.80.83.80:3131/admin?key=<secureEntrance>
```

后台登录密码来自：

```text
/opt/shaunx-blog/.env
```

字段：

```text
ADMIN_PASSWORD
```

后台支持：

- 新建、编辑、删除文章
- 上传和管理图片
- 编辑页面内容
- 修改站点配置

管理后台限制移动端访问，建议在桌面浏览器中使用。

```bash
  -C /opt/shaunx-blog shaunx-blog-data
```

备份代码和环境配置：

```bash
tar -czf /opt/shaunx-blog-app-backup-$(date +%Y%m%d-%H%M%S).tar.gz \
  -C /opt shaunx-blog
```

建议至少备份：

```text
/opt/shaunx-blog/shaunx-blog-data

```

`.env` 包含后台密码和 JWT 密钥，应妥善保存，不要公开。

## 卸载

### 仅停止服务，保留数据

```bash
cd /opt/shaunx-blog
docker compose --env-file .env -f docker/docker-compose.yml down
```

保留：

```text
/opt/shaunx-blog
/opt/shaunx-blog/shaunx-blog-data
```

### 删除应用代码，保留数据

由于数据目录位于项目目录内，删除代码前需要先把数据移出项目目录：

```bash
cd /opt/shaunx-blog
docker compose --env-file .env -f docker/docker-compose.yml down
mv /opt/shaunx-blog/shaunx-blog-data /opt/shaunx-blog-data-keep
rm -rf /opt/shaunx-blog
```

保留：

```text
/opt/shaunx-blog-data-keep
```

### 完全卸载

先备份：

```bash
tar -czf /opt/shaunx-blog-data-backup-$(date +%Y%m%d-%H%M%S).tar.gz \
  -C /opt/shaunx-blog shaunx-blog-data
```

停止并删除服务：

```bash
cd /opt/shaunx-blog
docker compose --env-file .env -f docker/docker-compose.yml down
```

删除代码和数据：

```bash
rm -rf /opt/shaunx-blog
```

可选：删除本地构建镜像：

```bash
docker image rm imliusx/shaunx-blog:latest
```

只在确认不再需要该镜像时执行。

## 常见问题

### 容器一直不是 healthy

检查日志：

```bash
cd /opt/shaunx-blog
docker compose --env-file .env -f docker/docker-compose.yml logs --tail=200 blog
```

检查本机访问：

```bash
curl -I http://127.0.0.1:3131
```

如果本机返回 `200 OK`，但 healthcheck 异常，检查镜像内是否有 `curl`，以及 compose 的 healthcheck 配置。

### 端口被占用

查看占用：

```bash
ss -ltnp | grep 3131
```

换端口：

```bash
nano /opt/shaunx-blog/.env
```

修改：

```env
BLOG_PORT=新端口
SITE_URL=http://服务器IP:新端口
```

重新启动：

```bash
cd /opt/shaunx-blog
docker compose --env-file .env -f docker/docker-compose.yml up -d
```

### 管理后台进不去

检查入口码：

```bash
grep secureEntrance /opt/shaunx-blog/shaunx-blog-data/config/site.config.json
```

检查后台密码：

```bash
grep ADMIN_PASSWORD /opt/shaunx-blog/.env
```

访问格式：

```text
http://23.80.83.80:3131/admin?key=<secureEntrance>
```

管理后台限制移动端访问，请使用桌面浏览器。

### 私有仓库无法拉取

服务器需要配置 GitHub Deploy Key。

检查 SSH：

```bash
ssh -T git@github.com
```

正常会显示类似：

```text
Hi imliusx/shaunx-blog! You've successfully authenticated, but GitHub does not provide shell access.
```

### 更新后样式或页面不对

重新构建：

```bash
cd /opt/shaunx-blog
docker compose --env-file .env -f docker/docker-compose.yml up -d --build
```

查看当前代码提交：

```bash
cd /opt/shaunx-blog
git log -1 --oneline
```
